# Contribution Guide

Thank you for your interest in contributing to ElasticDash! We welcome contributions of all kinds, including code, documentation, bug reports, and feature requests.

## How to Contribute

### 1. Fork the Repository
- Click the "Fork" button on the repository you want to contribute to (see Quick Links in the docs).

### 2. Clone Your Fork
- Clone your forked repository to your local machine:
  ```bash
  git clone https://github.com/<your-username>/<repo-name>.git
  ```

### 3. Create a Branch
- Create a new branch for your feature or bugfix:
  ```bash
  git checkout -b my-feature
  ```

### 4. Make Your Changes
- Make your changes, following the existing code style and conventions.
- Add or update tests as needed.
- Update documentation if your change affects usage or APIs.

### 5. Commit and Push
- Commit your changes with a clear message:
  ```bash
  git commit -m "Add my feature"
  git push origin my-feature
  ```

### 6. Open a Pull Request
- Go to your fork on GitHub and open a Pull Request (PR) to the main ElasticDash repository.
- Fill out the PR template and describe your changes clearly.

### 7. Review Process
- The maintainers will review your PR and may request changes or clarifications.
- Please respond to feedback and update your PR as needed.
- Once approved, your changes will be merged.

## Code of Conduct

By participating, you agree to follow our [Code of Conduct](CODE_OF_CONDUCT.md) and treat everyone with respect.

## Reporting Issues

- Use the GitHub Issues tab to report bugs or request features.
- Please provide as much detail as possible, including steps to reproduce bugs.

## Style Guidelines

- Follow the language-specific style guides (e.g., PEP8 for Python, Prettier/ESLint for JS/TS).
- Write clear, concise commit messages.
- Keep pull requests focused and minimal.

## Community

- Join discussions in GitHub Issues and Pull Requests.
- For major changes, open an issue first to discuss your proposal.

---

Thank you for helping make ElasticDash better!
